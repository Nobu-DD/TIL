# dockerfileとは
コンテナを構築するためのイメージを作成する設計図のようなもの。DockerHubに配布されているイメージをベースとし、自身の開発に合わせたイメージにカスタマイズする。<br>
## Dockerfileに書いていく内容
- FROM<br>
DockerHubから選んだベースとなるイメージを記述する。チーム開発の場合、使用するPCのOSが違うため、プロットフォームを指定する場合もある。(linux/amd64など)
- RUN<br>
docker buildでイメージを構築する際、シェルコマンドを実行するための命令文。ライブラリのインストールやセットアップの順番を指定するために使用する。
- CMD<br>
作成したコンテナを実行した時に発動させるコマンドを記述する。基本プログラミング言語の開発環境を立ち上げるコマンドを記述することが多い。(個人の主観)
- COPY<br>
ホスト上(ローカル)にあるディレクトリ構成をdockerイメージにコピーするために使用する。
- ENV<br>
dockerコンテナ内に環境変数を定義するために使用。
- WORKDIR<br>
dockerイメージを構築するために作業するディレクトリを指定。COPYやCMDなどを実行する際、少ない相対パスで指定することが出来る。
